# RNF-03. Documentación del sistema y soporte operativo

## Descripción

El sistema deberá contar con documentación técnica y de usuario clara, actualizada y accesible para facilitar su uso, mantenimiento y evolución.

La documentación deberá cubrir el funcionamiento general del sistema, el uso de sus principales módulos y la descripción técnica de los componentes necesarios para tareas de mantenimiento o futuras mejoras.

Asimismo, deberá incluir información suficiente tanto para el personal técnico como para los usuarios administrativos, permitiendo comprender la operación del sistema y consultar lineamientos de uso sin depender exclusivamente del conocimiento informal del equipo.

Como mínimo, la documentación deberá incluir una descripción general del sistema, documentación técnica de endpoints y componentes principales, instrucciones de uso para el personal administrativo y lineamientos o estándares de documentación aplicados.

El alcance de este requisito contempla la disponibilidad y calidad de la documentación del sistema, así como el soporte operativo básico para facilitar su consulta y aprovechamiento dentro del proyecto.

---

## Categoría

Mantenibilidad y soporte operativo

---

## Objetivo del requisito

Garantizar que el sistema disponga de documentación técnica y de usuario suficiente, clara y actualizada para facilitar su comprensión, operación, mantenimiento y evolución.

---

## Escenario de calidad

**Contexto:**  
Personal técnico o usuario administrativo necesita consultar información sobre el sistema web de la clínica.

**Estímulo:**  
Se requiere utilizar, mantener, revisar o actualizar algún módulo, componente o proceso del sistema.

**Respuesta esperada:** 
 
- La documentación está disponible y es accesible.
- Existe una descripción general del sistema y de sus módulos principales.
- El personal técnico puede consultar información sobre endpoints y componentes relevantes.
- El personal administrativo puede consultar instrucciones de uso del sistema.
- La documentación mantiene una estructura clara y consistente.

---

## Criterios de aceptación

- Existe documentación general que describe el propósito y funcionamiento básico del sistema.
- Existe documentación técnica de los endpoints y componentes principales del sistema.
- Existen instrucciones de uso dirigidas al personal administrativo.
- La documentación está disponible en un medio accesible para el equipo del proyecto.
- La documentación utiliza una estructura clara y consistente.
- La documentación cubre al menos los módulos principales del sistema.
- Los lineamientos o estándares de documentación aplicados están definidos.
- La información documentada corresponde con el funcionamiento actual del sistema.
- La documentación puede ser utilizada como apoyo para mantenimiento, capacitación o futuras mejoras.

---

## Verificación

El cumplimiento del requisito se verificará mediante:

- Revisión de existencia de documentación general del sistema.
- Revisión de documentación técnica de endpoints y componentes principales.
- Validación de instrucciones de uso para el personal administrativo.
- Comprobación de accesibilidad de la documentación para el equipo.
- Revisión de consistencia en formato, estructura y contenido.
- Comparación entre la documentación y el comportamiento actual del sistema.

---

## Consistencia

Este requisito es consistente con los requisitos funcionales relacionados con la gestión, consulta y administración de citas, ya que facilita la comprensión y uso correcto de los procesos implementados en el sistema.

También es consistente con los requisitos no funcionales de mantenibilidad, capacitación, soporte técnico y evolución del sistema, al proporcionar información útil para su operación y mejora continua.

No presenta contradicciones con otros requisitos del sistema.

---

## Métricas de cumplimiento

- El 100% de los módulos principales del sistema cuenta con documentación descriptiva.
- El 100% de los endpoints y componentes principales identificados cuenta con documentación técnica.
- El 100% de las funciones administrativas clave cuenta con instrucciones de uso.
- La documentación se encuentra accesible para el equipo del proyecto en un medio definido.
- La documentación refleja el funcionamiento actual del sistema en cada entrega revisada.
